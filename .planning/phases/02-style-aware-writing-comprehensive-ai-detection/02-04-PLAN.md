Create ai-patterns.md template as the pattern knowledge base:

**Structure:**

```markdown
# AI Language Pattern Database

## Built-In Patterns

### Pattern 1: Delve/Delved/Delving
**Category:** Word choice
**Why AI-ish:** Overly common in AI-generated text, rarely used in natural prose
**Detection:** Word forms: delve, delves, delved, delving
**Replacements:**
- "explored" (if examining something)
- "investigated" (if researching)
- "studied" (if learning)
- "examined" (if analyzing)
- Simply remove and restructure (often unnecessary)

### Pattern 2: Tapestry
**Category:** Metaphor overuse
**Why AI-ish:** AI loves "tapestry" for any complex interconnection
**Detection:** Word: tapestry (metaphorical usage)
**Replacements:**
- "web" or "network" (if connections)
- "mixture" or "blend" (if combination)
- "array" or "collection" (if variety)
- Specific concrete detail (better than metaphor)
- Simply remove and be direct

### Pattern 3: Navigate/Navigated
**Category:** Metaphor overuse
**Why AI-ish:** AI uses "navigate" metaphorically for any challenge
**Detection:** Navigate/navigated/navigating in non-literal contexts
**Replacements:**
- "faced" or "dealt with" (if challenges)
- "moved through" or "worked through" (if process)
- "handled" or "managed" (if situations)
- "explored" (if discovery)
- Be specific about the actual action

### Pattern 4: Testament To
**Category:** Phrase overuse
**Why AI-ish:** AI reaches for this phrase constantly
**Detection:** Phrase: "testament to"
**Replacements:**
- "proof of" or "evidence of"
- "shows" or "demonstrates"
- "reveals" or "indicates"
- Simply state the fact directly
- "example of" (if appropriate)

### Pattern 5: Juxtaposition
**Category:** Word choice
**Why AI-ish:** Overused in AI analysis and description
**Detection:** Word: juxtaposition
**Replacements:**
- "contrast" (simpler)
- "difference" (clearer)
- "comparison" (if comparing)
- "side by side" (more natural)
- Describe the actual relationship directly

### Pattern 6: Overly Formal Academic Language
**Category:** Tone
**Why AI-ish:** AI defaults to academic formality
**Detection patterns:**
- "Furthermore," "Moreover," "Additionally," at sentence starts
- "One might observe," "It can be seen that"
- "In conclusion," "To summarize" in narrative
- Latin phrases (e.g., "per se," "vis-à-vis") in casual prose
- "Aforementioned," "heretofore," "wherein"
**Replacements:**
- Use simpler conjunctions: "And," "But," "So"
- Direct statements instead of passive observation
- Remove conclusion markers in narrative
- Use English equivalents
- Modern plain language

### Pattern 7: "It's Worth Noting" Phrases
**Category:** AI hedging
**Why AI-ish:** AI loves these meta-commentary phrases
**Detection patterns:**
- "It's worth noting that"
- "It should be noted that"
- "Notably,"
- "Of note,"
- "Interestingly,"
**Replacements:**
- Simply state the fact directly
- "In fact," (if emphasis needed)
- Remove entirely (usually unnecessary)
- Let the interesting detail speak for itself

### Pattern 8: Purple Prose Patterns
**Category:** Style excess
**Why AI-ish:** AI can overdo literary flourishes
**Detection patterns:**
- Excessive adjective stacking (3+ adjectives)
- Melodramatic adverbs: "unutterably," "ineffably," "indescribably"
- Overwrought metaphors: "soul-crushing," "heart-wrenching" overuse
- Pretentious vocabulary where simple words work
- Every dialogue tag enhanced: "he breathed," "she whispered huskily"
**Replacements:**
- Reduce to 1-2 meaningful adjectives
- Remove or use simple adverbs
- Use grounded, specific detail instead of hyperbole
- Choose clear words over impressive ones
- Use "said" or "asked" for 80%+ of dialogue tags

## Custom Phrases

Writers can add their own overused words or phrases to watch for.

### How to Add Custom Phrases
Edit this section to add phrases you personally overuse or want to avoid:

```yaml
custom_patterns:
  - phrase: "suddenly"
    reason: "Overused as scene transition"
    alternatives: ["without warning", "in an instant", "restructure to show action directly"]

  - phrase: "very"
    reason: "Weak intensifier"
    alternatives: ["use stronger base word", "remove entirely", "specific description"]
```

Add entries following this format. Detection workflow will scan for these alongside built-in patterns.

**Notes:**

- Aim for natural, human voice appropriate to your story's style
- Some patterns may fit your genre (e.g., literary fiction vs thriller)
- Use STYLE.md preferences to guide replacement choices
```

Create `/write:detect` command workflow that scans prose for AI language patterns:

1. Validates story project exists (reads PROJECT.md)

2. Prompts writer for detection scope:
   - Option A: Single chapter (enter chapter number)
   - Option B: All chapters
   - Option C: Specific text (paste prose directly for one-time scan)

3. Loads pattern database:
   - Read .planning/templates/ai-patterns.md for 10 built-in patterns
   - Parse custom_patterns YAML section if exists
   - If project has stories/{slug}/AI-PATTERNS.md (project-specific copy), use that instead of template

4. Scans prose for each pattern:
   - Pattern 1-5, 7: Simple keyword/phrase matching
   - Pattern 6: Multiple phrase detection (formal academic language)
   - Pattern 8: Pattern-based detection (adjective stacking, melodramatic adverbs, etc.)
   - Custom patterns: Keyword matching from custom_patterns list

5. For each match found:
   - Record location: chapter number, approximate line/paragraph
   - Record matched text (surrounding context)
   - Record pattern category and why it's AI-ish
   - Load replacement suggestions from pattern database

6. Generates detection report:
   ```markdown
   # AI Pattern Detection Report
   **Story:** {title}
   **Scanned:** {scope description}
   **Patterns found:** {total count}

   ## Findings by Pattern

   ### Pattern 1: Delve/Delved (3 instances)

   **Chapter 2, paragraph 5:**
   > "She delved into the mystery surrounding the murder."

   **Why flagged:** Overly common in AI text, rarely natural
   **Suggestions:**
   - "She investigated the mystery surrounding the murder."
   - "She explored the mystery surrounding the murder."
   - Simply: "She looked into the murder mystery."

   [Repeat for each instance and pattern]

   ## Summary
   - Total flags: {N}
   - Most common: {pattern name} ({count} times)
   - Chapters affected: {list}

   ## Next Steps
   - Review flagged passages in context
   - Not all instances need changing - trust your judgment
   - Consider your STYLE.md preferences when choosing replacements
   - Re-run detection after revisions to verify cleanup
   ```

7. Save report to stories/{slug}/AI-DETECTION-REPORT.md with timestamp

8. Offer to create project-specific ai-patterns.md:
   - Copies template to stories/{slug}/AI-PATTERNS.md
   - Writer can customize patterns or add more custom phrases
   - Future detections use project copy instead of global template

9. Display summary stats and next steps

Implementation notes:
- Case-insensitive matching for keywords
- Word boundary detection (don't match "delve" inside "undelved")
- Pattern 6 & 8 are multi-pattern - check all sub-patterns
- Include line numbers or paragraph markers for easy location
- Replacements should match context (e.g., tense, formality from STYLE.md)

Conversational tone. Explain that detection helps awareness but writer makes final calls. Estimated time: 5-15 minutes depending on text volume.
