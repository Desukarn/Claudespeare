Enhance write-chapter.md workflow to calculate and update word counts:

**After chapter generation (in step where chapter is saved):**

1. Calculate chapter word count:
   - Count words in generated prose (exclude frontmatter, markdown formatting)
   - Display to writer: "Chapter {N} complete: {word_count} words"

2. Update chapter frontmatter word_count field

3. Read PROJECT.md frontmatter to get current_word_count

4. Calculate new total: current_word_count + chapter_word_count

5. Update PROJECT.md frontmatter current_word_count field

6. Display progress:
   ```
   Story progress: {new_total} / {goal} words ({percentage}%)
   Chapters: {count} | Average: {avg} words per chapter
   ```

**Before chapter generation (in prompting step):**

7. Check if writer has set chapter targets in PROJECT.md Progress section

8. If chapter target exists for this chapter number, display:
   ```
   Target for Chapter {N}: {target} words
   You can adjust during generation.
   ```

**Add helper instructions:**
- Word count is approximate (counts prose content, excludes formatting)
- Writer can manually adjust counts in PROJECT.md if needed
- Progress tracking helps maintain pacing and hit story length goals

Implementation notes:
- Use simple word count: split on whitespace, filter markdown symbols
- Update PROJECT.md by reading, modifying frontmatter, writing back
- Handle edge case where word_count_goal is null (show count but not percentage)
