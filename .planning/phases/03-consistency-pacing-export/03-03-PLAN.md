Create `/write:export` command workflow (200+ lines) with the following structure:

**Overview section** (what it does):
- Exports individual chapters or compiles full manuscript
- Supports 3 formats: Markdown (.md), Plain text (.txt), Standard manuscript (.txt with industry formatting)
- Generates professional title pages
- Applies proper formatting (headers, scene breaks, spacing)
- Outputs to exports/ directory in story project
- 5-15 minute execution time

**Prerequisites check**:
- Valid story project exists
- At least 1 chapter written (chapters/chapter-*.md)
- PROJECT.md exists for metadata

**Workflow steps**:

Step 1: Prompt for export parameters
Ask writer:
- **Export mode:**
  - `single`: Export one chapter
  - `range`: Export chapter range (e.g., 1-5)
  - `full`: Export complete manuscript
- **Format:**
  - `markdown`: Preserve markdown formatting (.md)
  - `text`: Clean plain text (.txt)
  - `manuscript`: Standard manuscript format (.txt with industry conventions)
- **Include title page:** Yes/No (only for full/range exports)

If `single`: Ask for chapter number
If `range`: Ask for start and end chapter numbers
If `full`: Automatically detect all chapters

Step 2: Load story metadata (OUT-01, OUT-02)
Read PROJECT.md:
- Extract story title (from `# {TITLE}` heading)
- Extract genre (from Metadata section)
- Extract current word count (or calculate from chapters)
- Prompt for author name if not in PROJECT.md
- If title page requested, prompt for optional contact info (email, phone, address)

Read chapter files in sequence:
- For `single`: Read chapters/chapter-{N}.md
- For `range`: Read chapters/chapter-{start} through chapters/chapter-{end}.md
- For `full`: Read all chapters/chapter-*.md in numeric order
- Parse each chapter:
  - Extract frontmatter (chapter number, title, word_count)
  - Extract prose content (everything between frontmatter and ## Notes section)
  - Preserve scene breaks (--- markers)
  - Strip chapter metadata frontmatter from exported content

Step 3: Generate title page (OUT-03)
If title page requested, use manuscript-title-page.md template:

Replace template variables:
- `{word_count}`: Total word count of exported content
- `{genre}`: Genre from PROJECT.md
- `{TITLE}`: Story title in ALL CAPS
- `{AUTHOR}`: Author name (from prompt or PROJECT.md)
- `{author_email}`, `{author_phone}`, `{author_address}`: Contact info if provided, otherwise omit

**For markdown format:**
- Use markdown heading formatting
- Preserve template spacing

**For text format:**
- Convert to plain text with spacing preservation
- Center-align title, "by", and author name using spaces

**For manuscript format:**
- Plain text with exact spacing (industry standard)
- Word count upper right
- Title centered ~1/3 down page
- Contact info lower left

Step 4: Compile chapters
Concatenate chapters in numeric order:

**For single chapter:**
- Just export that chapter's content

**For range/full:**
- Combine chapters sequentially
- Add chapter headers between chapters:
  - Markdown format: `# Chapter {N}: {Title}` (or `# Chapter {N}` if no title)
  - Text format: `Chapter {N}: {Title}` with blank line after
  - Manuscript format: New page for each chapter with `{Author Last Name} / {Story Title} / {Page #}` header

**Scene breaks:**
- Markdown: Preserve `---` (horizontal rule)
- Text: Convert to `* * *` (standard scene break)
- Manuscript: Use `#` centered (industry standard scene break)

Step 5: Apply format-specific formatting (OUT-04)

**Markdown format (.md):**
- Preserve all markdown formatting
- Keep chapter headings as H1 (#)
- Preserve scene breaks as ---
- Keep paragraph structure
- Output: `exports/{story-slug}-{mode}-{timestamp}.md`

**Plain text format (.txt):**
- Strip all markdown syntax (headings, bold, italic, links)
- Convert heading markers to plain text
- Scene breaks become `* * *`
- Preserve paragraph breaks (double newline)
- Clean readable text suitable for any platform
- Output: `exports/{story-slug}-{mode}-{timestamp}.txt`

**Standard manuscript format (.txt):**
- Industry-standard formatting conventions:
  - 12pt monospace equivalent (plain text, no actual font control)
  - Double-spaced (blank line between paragraphs)
  - 1-inch margins (65-70 characters per line for text file)
  - Headers on each page: `{LAST NAME} / {TITLE} / {PAGE}`
  - Page breaks between chapters
  - Scene breaks as single `#` centered
  - No bold, italic, or special formatting
  - First line of paragraphs NOT indented (submission standard)
- Output: `exports/{story-slug}-manuscript-{timestamp}.txt`

Step 6: Calculate final word count
Count words in exported content:
- Split on whitespace
- Exclude frontmatter and markdown symbols
- Display total to writer

Step 7: Write export file
Create exports/ directory if it doesn't exist:
```bash
mkdir -p stories/{slug}/exports
```

Write compiled content to file:
- Full manuscript: `{story-slug}-full-manuscript.{ext}`
- Chapter range: `{story-slug}-chapters-{start}-{end}.{ext}`
- Single chapter: `{story-slug}-chapter-{N}.{ext}`

Add timestamp suffix for version tracking: `-{YYYY-MM-DD-HHMM}`

Step 8: Confirm export
Display to writer:
- File path of exported manuscript
- Final word count
- Format used
- Chapter count or range
- Suggest next steps:
  - Review in text editor
  - Run spell check
  - Submit to platform or send to beta readers

**Examples section**:
Show 3 complete export scenarios:

Example 1: Single chapter markdown export
- Mode: single, chapter 5
- Format: markdown
- Output: `exports/my-story-chapter-5-2026-02-27-1430.md`
- Content preview showing chapter heading and first paragraph

Example 2: Full manuscript plain text export
- Mode: full (10 chapters)
- Format: text
- Title page: Yes
- Output: `exports/my-story-full-manuscript-2026-02-27-1430.txt`
- 25,000 words

Example 3: Standard manuscript format for submission
- Mode: full
- Format: manuscript
- Title page: Yes with contact info
- Output: `exports/my-story-manuscript-2026-02-27-1430.txt`
- Showing header format, page structure, scene breaks

**FAQ section**:
- Q: "What's the difference between text and manuscript format?" A: Text is clean readable plain text; manuscript follows industry submission standards with headers, spacing, and scene break conventions
- Q: "Can I export to Word or PDF?" A: Not yet - export to .txt and import into Word/Google Docs, then format and export PDF from there
- Q: "What if some chapters are missing?" A: Export will skip missing chapter numbers and warn you
- Q: "Can I re-export after edits?" A: Yes, run `/write:export` again - timestamps prevent overwriting
- Q: "Should I include title page for beta readers?" A: Optional - title page is mainly for formal submissions

Add usage notes section:

```markdown
## Usage Notes

This template follows standard manuscript submission conventions:

- **Word count:** Upper right corner
- **Genre:** Below word count (optional)
- **Title:** Centered, approximately 1/3 down the page (12-15 blank lines from top)
- **By-line:** Centered below title
- **Author:** Centered below "by"
- **Contact:** Lower left corner (3-5 lines from bottom)
- **No header or page number** on title page

## Variable Replacement

During export, these placeholders are replaced:

- `{word_count}`: Total manuscript word count
- `{genre}`: Genre from PROJECT.md
- `{TITLE}`: Story title in ALL UPPERCASE
- `{AUTHOR}`: Author name (prompted or from PROJECT.md)
- `{author_email}`: Email address (optional)
- `{author_phone}`: Phone number (optional)
- `{author_address}`: Mailing address (optional)

Contact fields are omitted if not provided.

## Format Variations

**Markdown export:**
Uses markdown heading syntax with proper spacing.

**Text/Manuscript export:**
Plain text with spacing preserved using blank lines and approximate centering.

## Alternative Minimal Format

For less formal submissions or beta readers:

```
{TITLE}
by {AUTHOR}
{word_count} words
```

Writers can customize this template per project by creating a copy in their story directory.
```
