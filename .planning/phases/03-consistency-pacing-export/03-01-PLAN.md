  <files>.claude/skills/write/check-consistency.md</files>
  <action>
Create `/write:check` command workflow (200+ lines) with the following structure:

**Overview section** (what it does):
- Scans chapters against established story context (CHARACTERS, WORLD, chapters)
- Detects 5 consistency violation types: character details, character voice, timeline, world rules, POV shifts
- Generates detailed report with locations and specific issues
- 5-15 minute execution time for full story scan

**Prerequisites check**:
- Valid story project exists
- At least 1 chapter written (chapters/chapter-*.md)
- At least 1 context file exists (CHARACTERS.md or WORLD.md)

**Workflow steps**:

Step 1: Load story context
- Read CHARACTERS.md (or yolo-characters.md): Extract for each character:
  - Physical description (appearance details like eye color, hair, height, scars)
  - Voice/speech patterns (tone, vocabulary, distinctive phrases)
  - Personality traits (consistent behaviors)
  - Background facts (history, relationships, occupation)
- Read WORLD.md (if exists): Extract:
  - World Rules > Key Rules (magic system, tech limitations)
  - World Rules > Limitations (what can't happen)
  - Consistency Tracking > Details to Maintain (established facts)
  - Consistency Tracking > Established Facts (facts from prior chapters)
- Read all existing chapters (chapters/chapter-*.md): Build chapter sequence with POV tracking

Step 2: Scan for character detail inconsistencies (CONS-01, CONS-02)
For each chapter:
- Parse character mentions in prose
- Check physical descriptions against CHARACTERS.md baseline
  - Flag: Eye color changes, hair color/style changes, height contradictions, missing/appearing scars
  - Example: "Chapter 3 describes blue eyes but Chapter 1 established brown eyes"
- Check voice consistency against CHARACTERS.md patterns
  - Flag: Formal character suddenly using slang, vocabulary level shifts, distinctive phrase abandonment
  - Example: "Character speaks casually in Ch 5 but CHARACTERS.md established formal speech reflecting magical training"
- Check personality consistency
  - Flag: Character acts against established traits without explanation
  - Example: "Stubborn character gives up easily in Ch 7 without character growth justification"

Step 3: Scan for timeline and continuity errors (CONS-03)
- Build timeline from chapter Notes sections and prose
- Check for:
  - Temporal contradictions (event A happens after B, then later B happens after A)
  - Missing time (character teleports between distant locations unrealistically)
  - Seasonal inconsistencies (winter -> summer -> winter in 3 days)
  - Age contradictions (character stated as 25 in Ch 1, 30 in Ch 4 with no time passage)
  - Event order violations (character knows something before they learn it)
- Flag with: Chapter references, timeline reconstruction, conflict description

Step 4: Scan for world rule violations (CONS-04)
- Check prose against WORLD.md rules:
  - Magic/tech use violates stated limitations (e.g., "rule says can't read minds, but Ch 6 has mind reading")
  - Costs/requirements ignored (e.g., "magic drains vitality but character uses it repeatedly with no fatigue")
  - Established facts contradicted (e.g., "travel takes 5 days per WORLD.md but character travels in 1 day with no explanation")
  - Geography contradictions
  - Cultural/social rule violations without intentional character choice
- Flag with: Rule quoted, chapter location, violation description

Step 5: Scan for POV inconsistencies (CONS-05)
- Parse chapter frontmatter `pov:` field
- Scan prose for POV violations:
  - Head-hopping within scenes (accessing non-POV character thoughts without scene break)
  - Narrator knowing what POV character cannot know
  - Perspective shifts (first-person to third-person mid-scene)
- Flag with: Chapter/paragraph location, POV established, violation quote

Step 6: Generate CONSISTENCY-REPORT.md
Using consistency-report.md template, populate:
- Summary statistics (total chapters scanned, total flags, breakdown by type)
- Section per violation type with findings
- Each finding includes:
  - Location (chapter number, approximate paragraph/line if possible)
  - Issue description (what's inconsistent)
  - Baseline reference (what was established in CHARACTERS/WORLD)
  - Suggested fix or question to writer
- Severity indicators: Critical (major contradiction), Warning (minor inconsistency), Note (verify if intentional)

**Examples section**:
Show 2-3 example violations with full context:
- Character detail contradiction example
- World rule violation example
- POV shift example

**FAQ section**:
- Q: "What if inconsistency is intentional character growth?" A: Review report, mark intentional changes in ARCS.md
- Q: "Can I customize what gets checked?" A: Yes, update CHARACTERS.md or WORLD.md baseline details
- Q: "How often should I run this?" A: After every 3-5 chapters or before major revisions

  <name>Task 2: Create consistency report template</name>

Create structured template for consistency reports (50+ lines):

**Frontmatter** (YAML):
```yaml
---
story: {project slug}
checked: {timestamp}
chapters_scanned: {number}
total_flags: {count}
critical: {count}
warnings: {count}
notes: {count}
---
```

**Summary section**:
- Overview of scan (chapters, time period)
- Total flags by severity
- Total flags by type (character, timeline, world, POV)
- Most common issue type

**Character Consistency section**:
Template for character detail violations:
```
### Character: {Name}

**Issue:** {Description}
**Location:** Chapter {N}, {scene/paragraph descriptor}
**Baseline:** {What CHARACTERS.md established}
**Violation:** {What chapter says}
**Severity:** {Critical/Warning/Note}
**Suggestion:** {Possible fix or question}
```

**Timeline & Continuity section**:
Template for timeline violations:
```
### {Event/Sequence}

**Issue:** {Temporal contradiction}
**Chapters:** {N, M, P}
**Timeline conflict:** {Description}
**Severity:** {Critical/Warning}
**Suggestion:** {Reorder events or clarify time passage}
```

**World Rules section**:
Template for world rule violations:
```
### Rule: {Rule name from WORLD.md}

**Issue:** {Violation description}
**Location:** Chapter {N}
**Rule states:** {Quote from WORLD.md}
**Chapter text:** {Quote from chapter}
**Severity:** {Critical/Warning}
**Suggestion:** {Fix or justify exception}
```

**POV Consistency section**:
Template for POV violations:
```
### Chapter {N}: {Scene}

**Issue:** {POV shift/head-hopping}
**Established POV:** {Character from frontmatter}
**Violation:** {Quote showing perspective shift}
**Location:** {Scene/paragraph}
**Severity:** {Warning/Note}
**Suggestion:** {Add scene break or revise to maintain POV}
```

**Next Steps section**:
- Review each flagged issue
- Mark intentional choices in ARCS.md or WORLD.md
- Update baseline files if details evolved
- Revise chapters to fix critical contradictions
- Run `/write:check` again after fixes

**Notes section**:
- Empty section for writer annotations
- Space to mark "fixed", "intentional", "ignore"
