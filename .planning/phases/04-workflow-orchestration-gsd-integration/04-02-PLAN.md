  <name>Task 1: Create adaptive discussion workflow</name>
  <files>.claude/skills/write/discuss-phase.md</files>
  <action>
Create `/write:discuss-phase N` workflow that implements adaptive questioning based on phase type and goals.

**Core Workflow:**
1. **Validate Context:**
   - Check that story project exists (read PROJECT.md)
   - Verify phase number is valid (1-5 typical range)
   - Load existing context from PHASE-{N}-CONTEXT.md if exists (resume discussion)

2. **Phase Detection:**
   Based on phase number and project state, determine phase type:
   - Phase 1: Planning/Setup (characters, outline, world-building)
   - Phase 2+: Writing/Drafting (chapters, scenes, revisions)

3. **Adaptive Questions:**

   **For Planning Phases (1):**
   - "What's your approach for this phase?" (YOLO expand vs In-Depth planning)
   - "Which story elements will you develop?" (characters, plot, world, arcs, style)
   - "How much detail do you want?" (light sketches vs comprehensive)
   - "Any specific concerns or things to explore?"

   **For Writing Phases (2+):**
   - "Which chapters will you write in this phase?" (chapter numbers or ranges)
   - "What plot beats from your outline will these cover?"
   - "Are there any character arcs to focus on?"
   - "Do you want to adjust pacing or style for these chapters?"
   - "Any consistency concerns from previous chapters?"
   - "Will you run /write:detect for AI language after writing?"

   **For Revision Phases:**
   - "Which chapters need revision?"
   - "What issues did you identify?" (plot holes, inconsistencies, pacing)
   - "Will you use /write:check for consistency validation?"
   - "Are you ready for export, or more drafting needed?"

4. **Capture Decisions:**
   Create PHASE-{N}-CONTEXT.md in story project directory:
   ```markdown
   ---
   phase: {N}
   phase_type: {planning|writing|revision}
   created: {timestamp}
   ---

   # Phase {N} Context

   ## Goals

   {Bulleted list of what writer wants to accomplish}

   ## Decisions

   {Bulleted list of explicit decisions made}

   ## Focus Areas

   {What to prioritize - characters, chapters, arcs, etc.}

   ## Concerns

   {Optional - anything to watch for or address}

   ## Next Steps

   1. {Specific actionable next steps based on discussion}
   2. {e.g., "Run /write:chapter 1", "Run /write:expand"}
   ```

5. **Workflow Orchestration Guidance:**
   After discussion, provide clear next steps:

   **For Planning:**
   - "Next: Run /write:expand (YOLO) or /write:plan (In-Depth)"
   - "Then: Run /write:discuss-phase 2 when ready to start writing"

   **For Writing:**
   - "Next: Run /write:chapter {N} to start drafting"
   - "After 2-3 chapters: Consider running /write:detect to check for AI language"
   - "After 5+ chapters: Run /write:check for consistency validation"
   - "When phase complete: Run /write:discuss-phase {N+1} to plan next batch"

   **For Revision:**
   - "Next: Make revisions based on consistency report"
   - "Then: Run /write:export when ready to compile manuscript"

6. **Resume Support:**
   - If PHASE-{N}-CONTEXT.md exists, offer to review and update decisions
   - Allow writer to refine goals without starting over
   - Preserve previous decisions with update timestamps

**Presentation Style:**
- Use numbered questions for clear flow
- Show current project context at start (title, genre, current word count)
- Format options clearly with bullets
- Use **bold** for emphasis on key choices
- Provide examples where helpful
- Keep language encouraging and supportive

**Estimated Duration:** 10-20 minutes depending on phase complexity

```markdown
**Typical Flow:**
1. `/write:new-project` - Initialize with guided setup
2. `/write:discuss-phase 1` - Clarify planning approach
3. `/write:expand` (YOLO) or `/write:plan` (In-Depth) - Develop story foundation

5. `/write:discuss-phase 2` - Plan writing phase (which chapters, focus areas)
6. `/write:chapter N` - Draft chapters one by one
7. `/write:detect` - Check for AI language patterns
8. `/write:check` - Validate consistency
9. `/write:discuss-phase 3` - Plan revisions if needed
10. `/write:export` - Compile finished manuscript

Repeat steps 5-8 for additional writing phases as story develops.
```
