  <action>
Create character bank workflow with three commands: `/write:character`, `/write:update-character`, and automatic bank population.

**1. Character Bank Structure:**

Characters stored in `.writing/characters/` directory (in story project root).

Each character file: `.writing/characters/{character-slug}.md`

Format:
```markdown
---
name: {Full Name}
slug: {character-slug}
role: {protagonist|antagonist|supporting}
first_appearance: chapter-{N}
last_updated: {timestamp}
---

# {Character Name}

## Core Identity

**Role:** {protagonist/antagonist/supporting}
**Age:** {age or range}
**Occupation:** {job/role in story}

## Physical Appearance

{Detailed description - height, build, hair, eyes, distinguishing features, style}

**Key Visual:** {1-2 sentence memorable visual that appears in prose}

## Personality

**Traits:** {3-5 key traits}
**Quirks:** {mannerisms, habits, speech patterns}
**Voice:** {how they speak - formal/casual, verbose/terse, any catchphrases}

## Background

{Brief history relevant to story}
{Formative experiences}
{Relationships to other characters}

## Goals and Motivations

**What they want:** {external goal}
**Why they want it:** {internal motivation}
**What stands in their way:** {obstacles}

## Character Arc

**Starting point:** {who they are at story start}
**Transformation:** {how they change}
**Ending point:** {who they become}

## Consistency Notes

**Appearance locks:** {details that must never change - eye color, scars, height}
**Voice markers:** {consistent speech patterns}
**Behavior patterns:** {how they react to stress, conflict, etc.}

## Chapter References

- Chapter 1: {brief note on what happened with this character}
- Chapter 3: {brief note}
- ...
```

**2. `/write:character <name>` - Query Character:**

1. Validate story project exists
2. Look up character in `.writing/characters/{slug}.md`
3. If found: display full profile
4. If not found:
   - Check CHARACTERS.md for character
   - Offer to create bank entry from CHARACTERS.md data
   - Or offer to create new character entry

**3. `/write:update-character <name>` - Update Character:**

1. Load existing character file from `.writing/characters/{slug}.md`
2. Display current profile
3. Ask what to update:
   - Physical appearance (add/change details)
   - Personality traits
   - Background information
   - Goals/motivations
   - Voice markers
   - Consistency locks
   - Chapter references (add notes about character in new chapters)
4. Update file with timestamp
5. Preserve all other sections unchanged

**4. `/write:populate-bank` - Auto-Populate from CHARACTERS.md:**

Run during project initialization or on-demand:

1. Read CHARACTERS.md or yolo-characters.md
2. For each character listed, create `.writing/characters/{slug}.md`
3. Extract information from CHARACTERS.md into bank format
4. Initialize empty sections (Chapter References, Consistency Notes)
5. Report what was created

**5. Integration with Chapter Generation:**

Update write-chapter.md to include:

Before generating chapter prose:
```
Load character bank:
1. List files in .writing/characters/
2. For each POV character and major character in scene, read their profile
3. Include character bank data in generation context
4. Specifically reference: voice markers, appearance locks, behavior patterns
```

After chapter generation:
```
Update character bank:
1. For characters who appeared in chapter, add brief note to Chapter References section
2. Flag if any character detail might need updating based on chapter events
```

Create chapter version control workflow with revision tracking, listing, restoration, and diffing.

**1. Revision Storage Structure:**

Revisions stored in `.writing/chapters/{chapter-slug}/revisions/` directory.

Each revision: `.writing/chapters/{chapter-slug}/revisions/v{NN}-{timestamp}.md`

Latest version: `.writing/chapters/{chapter-slug}/current.md` (or chapters/chapter-{N}.md in project root)

Revision metadata: `.writing/chapters/{chapter-slug}/revisions/index.json`

```json
{
  "chapter": "chapter-1",
  "title": "The Beginning",
  "revisions": [
    {
      "version": 1,
      "timestamp": "2026-02-27T10:30:00Z",
      "word_count": 2340,
      "note": "Initial draft"
    },
    {
      "version": 2,
      "timestamp": "2026-02-28T14:15:00Z",
      "word_count": 2450,
      "note": "Added opening scene detail"
    }
  ],
  "current_version": 2
}
```

**2. Automatic Revision Saving:**

When chapter is updated (via /write:chapter regeneration or manual edit):

1. Before overwriting, copy current version to revisions/
2. Increment version number
3. Calculate word count
4. Update index.json with metadata
5. Prompt for optional revision note

**3. `/write:revisions <chapter-name>` - List Revision History:**

1. Validate story project exists
2. Look up `.writing/chapters/{chapter-slug}/revisions/index.json`
3. Display revision history in table format:

```
Revisions for Chapter 1: The Beginning

| Ver | Date       | Time  | Words | Note                        |
|-----|------------|-------|-------|-----------------------------|
| 2   | 2026-02-28 | 14:15 | 2450  | Added opening scene detail  |
| 1   | 2026-02-27 | 10:30 | 2340  | Initial draft               |

Current: Version 2 (2450 words)
```

4. Offer actions: restore, diff, view specific version

**4. `/write:restore <chapter-name> <version-number>` - Restore Previous Version:**

1. Load specified revision from `.writing/chapters/{chapter-slug}/revisions/v{NN}-*.md`
2. Display preview (first 500 characters)
3. Confirm: "Restore chapter to version {N}? Current version will be saved as a new revision."
4. If confirmed:
   - Save current version as new revision
   - Copy specified revision to current location
   - Update index.json
5. Display confirmation with new version number

**5. `/write:diff <chapter-name> <version1> <version2>` - Compare Versions:**

1. Load both versions from revisions directory
2. Perform basic text comparison:
   - Show word count change
   - Highlight major additions/deletions (paragraph level)
   - Show metadata differences
3. Display side-by-side or unified diff format (human-readable, not raw diff)

Example output:
```
Comparing Chapter 1: Version 1 vs Version 2

Word Count: 2340 → 2450 (+110 words)

Changes:
+ Added paragraph 2: "The morning mist clung to the valley..."
+ Added dialogue exchange in scene 3
~ Modified ending paragraph (minor edits)
- Removed transitional sentence in scene 2
```

**6. Integration with Chapter Writing:**

Update write-chapter.md to include revision saving:

After generating new chapter content:
```
Save revision:
1. Check if chapter file already exists
2. If exists, save current version to revisions/ before overwriting
3. Update index.json with new revision metadata
4. Prompt for optional note about changes made
```

**Add command entries:**

```markdown
### `/write:character <name>`
View character profile from persistent bank

Query the character bank to view full profile for any character. Character bank lives in .writing/characters/ and persists across all chapters. If character not in bank yet, offers to populate from CHARACTERS.md or create new entry. Use this to reference character details before writing scenes to ensure consistency.

See: character-bank.md

### `/write:update-character <name>`
Update character details in bank

Modify character profile in the bank: appearance, personality, voice markers, consistency locks, background, goals. Updates timestamp and preserves existing information. Use after chapters where character develops or new details are revealed.

See: character-bank.md

### `/write:revisions <chapter-name>`
View chapter revision history

Display all saved versions of a chapter with timestamps, word counts, and notes. Shows version number, date, time, word count, and revision notes. Use this to track chapter development and decide if rollback is needed.

See: chapter-versions.md

### `/write:restore <chapter-name> <version>`
Restore previous chapter version

Roll back a chapter to a previous version. Shows preview before restoring. Current version is automatically saved as a new revision so no work is lost. Use when a revision didn't work out and you want to return to an earlier draft.

See: chapter-versions.md

### `/write:diff <chapter-name> <v1> <v2>`
Compare two chapter versions

View differences between two versions of a chapter. Shows word count changes and paragraph-level additions/deletions in human-readable format. Use to understand what changed between revisions.

See: chapter-versions.md
```
