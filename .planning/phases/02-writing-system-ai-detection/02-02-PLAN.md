Create comprehensive AI language pattern library (.planning/templates/ai-detection-patterns.md) with detection rules and human-sounding replacements:

**Structure for each pattern category:**

1. **Overused AI Verbs**
   - Pattern: "delve/delved/delving"
     - Detection regex: `\b(delve|delved|delving)\b`
     - Why flagged: "Rarely used in natural speech, common AI writing marker"
     - Replacements: explore, investigate, examine, look into, dig into, study
     - Example: "She delved into the mystery" → "She investigated the mystery"

   - Pattern: "navigate/navigated/navigating"
     - Detection regex: `\b(navigate|navigated|navigating)\b`
     - Why flagged: "Overused metaphorically by AI, sounds corporate"
     - Replacements: work through, deal with, face, handle, manage, confront
     - Example: "He navigated the challenges" → "He faced the challenges"

2. **AI Metaphors**
   - Pattern: "tapestry"
     - Detection regex: `\btapestry\b`
     - Why flagged: "Cliché AI metaphor for complexity"
     - Replacements: mix, blend, combination, collection, array, web
     - Example: "A tapestry of emotions" → "A mix of emotions"

   - Pattern: "testament to"
     - Detection regex: `\btestament to\b`
     - Why flagged: "Overly formal, AI prose marker"
     - Replacements: proof of, shows, demonstrates, evidence of, sign of
     - Example: "Testament to her skill" → "Proof of her skill"

3. **Formal Academic Patterns**
   - Pattern: "However, it is important to note that"
     - Detection regex: `\b(however|furthermore|moreover|nevertheless|thus|hence|therefore), it is (important|worth noting|imperative|crucial) (to note that|that)\b`
     - Why flagged: "Academic writing style, unnatural in fiction"
     - Replacements: But, And, Still, Yet, So (simpler transitions)

   - Pattern: "In order to"
     - Detection regex: `\bin order to\b`
     - Why flagged: "Unnecessarily formal"
     - Replacements: to (just use 'to')
     - Example: "In order to escape" → "To escape"

   - Pattern: "It is worth noting that"
     - Detection regex: `\bit is worth noting that\b`
     - Why flagged: "Academic voice, not fiction"
     - Replacements: Note that, Notably, (or remove entirely)

4. **Passive Construction Overuse**
   - Pattern: "was [verb]ed by"
     - Detection regex: `\bwas \w+ed by\b`
     - Why flagged: "Passive voice weakens fiction"
     - Guidance: "Rewrite in active voice"
     - Example: "The door was opened by John" → "John opened the door"

5. **Redundant Intensifiers**
   - Pattern: "very/really/quite/rather + adjective"
     - Detection regex: `\b(very|really|quite|rather) (good|bad|big|small|interesting|important)\b`
     - Why flagged: "Weak intensifiers, use stronger adjectives"
     - Replacements: Replace with stronger single adjective
     - Example: "very big" → "enormous", "really good" → "excellent"

6. **AI Sentence Starters**
   - Pattern: "It's important to note/understand/remember that"
     - Detection regex: `\bit's (important|crucial|worth noting|imperative) to (note|understand|remember) that\b`
     - Why flagged: "Explanatory AI voice, not storytelling"
     - Replacements: Start directly with the fact

**Pattern Summary Table:**
Create a quick reference table listing all patterns with:

- Category (verb/metaphor/formal/passive/intensifier/starter)
- Severity (high/medium/low)

**Usage Instructions:**
- How to scan text for these patterns
- When to apply detection (post-generation step)
- How to evaluate replacement suggestions

Create AI detection workflow document (.planning/templates/ai-detection-workflow.md) that guides writers through the detection and revision process:

**Workflow Overview:**
Step-by-step process for running AI detection on generated chapters/scenes

**When to Run Detection:**
- After generating any chapter or scene
- During revision passes
- Before marking content as final

**Detection Process:**

1. **Scan Text**
   - Load the pattern library: @.planning/templates/ai-detection-patterns.md
   - Read through generated content
   - Identify flagged patterns manually or via search
   - Mark each instance with line/paragraph number

2. **Review Flagged Patterns**
   - For each flagged instance:
     - Read the pattern explanation (why it's flagged)
     - Review suggested replacements
     - Consider story context and character voice
     - Decide: replace, rewrite, or keep (with justification)

3. **Apply Revisions**
   - Replace obvious AI patterns first (delve → investigate)
   - Rewrite passive constructions in active voice
   - Simplify formal academic language
   - Strengthen weak intensifiers with better adjectives
   - Remove unnecessary sentence starters

4. **Verify Character Voice**
   - Check that replacements match character speech patterns (from CHARACTERS.md)
   - Ensure revised text maintains character personality
   - Verify dialogue still sounds like the specific character

5. **Re-scan for Missed Patterns**
   - Do a second pass after revisions
   - Check for patterns introduced during rewriting
   - Confirm all high-severity patterns addressed

**Detection Checklist:**
Create a checklist for each chapter/scene:
- [ ] Scanned for all AI verb patterns (delve, navigate, etc.)
- [ ] Checked for AI metaphors (tapestry, testament)
- [ ] Identified formal academic language
- [ ] Flagged passive constructions
- [ ] Reviewed weak intensifiers
- [ ] Replaced or justified each flagged instance
- [ ] Verified character voice consistency
- [ ] Re-scanned after revisions

**Integration with Chapter/Scene Templates:**
- Add "Run AI Detection" to post-generation checklist in chapter.md
- Add "Run AI Detection" to post-generation checklist in scene.md
- Reference this workflow from those templates

**Example Detection Session:**
Show a concrete example:
- Sample paragraph with AI patterns
- Flagged instances with pattern names
- Replacement suggestions applied
- Final revised paragraph
