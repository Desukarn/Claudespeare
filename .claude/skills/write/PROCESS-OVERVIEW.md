## YOLO Mode Complete Workflow

**Philosophy:** Fast drafting with quality safeguards. Minimal upfront planning, discover through writing, BUT prevent cookie-cutter patterns.

**Time to First Draft:** ~2 hours for 80k words

### YOLO Mode Quality Checkpoints

**Total Mandatory Checkpoints: 3**

1. **Outline Approval (Step 2.7)**
   - Cannot skip
   - Requires explicit "YES"
   - Catches exposition dumps in planning phase
   - Verifies Goal/Conflict/Outcome for all chapters

2. **Pre-Writing Scene Analysis (Step 3.6.5)**
   - Automatic, runs before writing
   - Pauses if 3+ issues detected
   - Verifies scene structure
   - Flags passive patterns

3. **Post-Generation Structural Review (Step 3.9.5)**
   - Automatic, runs after writing
   - Pauses if 3+ issues detected
   - Offers auto-revision
   - Catches slipped-through patterns

### In-Depth Mode Quality Checkpoints

**Total Quality Checkpoints: 3 per arc**

Per Arc:
1. **Arc Planning (/write:discuss)**
   - Interactive questions
   - Creates arc context

2. **Pre-Writing Scene Analysis**
   - Same as YOLO (Step 3.6.5)
   - Automatic before writing

3. **Post-Generation Structural Review**
   - Same as YOLO (Step 3.9.5)
   - Automatic after writing

**Difference from YOLO:** User has checkpoint BETWEEN arcs to review and revise before continuing.

## Quality Checkpoints Summary

### All Mandatory Checkpoints

| Checkpoint | When | Mode | Skippable? | Purpose |
|------------|------|------|------------|---------|
| **Outline Approval** | After outline generation | YOLO only | NO | Catch exposition dumps in planning |
| **Pre-Writing Scene Analysis** | Before spawning writers | Both | NO (pauses if issues) | Verify scene structure |
| **Chapter Writer Checklist** | Before each chapter | Both | NO | Commit to quality standards |
| **Post-Gen Structural Review** | After chapters written | Both | NO (pauses if issues) | Catch slipped patterns |
| **Arc Planning** | Between arcs | In-Depth only | NO | Plan next arc details |

### What Each Checkpoint Catches

**Outline Approval:**
- Passive opening (no immediate conflict)
- Tavern/inn exposition scenes
- "Character learns about" chapters
- Lack of Goal/Conflict/Outcome
- Protagonist passivity
- Story starting too early/late

**Pre-Writing Scene Analysis:**
- Scene-level exposition dumps
- Passive information gathering
- Convenient conversations
- Missing conflict in scenes
- Info delivery without cost

**Chapter Writer Checklist:**
- Active commitment to quality
- Show vs Tell awareness
- Exposition dump prevention
- Character agency focus
- Prose quality standards

**Post-Generation Review:**
- Actual exposition dump text
- Passive scene patterns
- Convenient NPC conversations
- Filter words and telling
- AI language patterns

## Exposition Dump Prevention System

### Multi-Layer Defense

**Layer 1: Planning Phase (Outline)**
- Automatic keyword detection
- Pattern recognition
- Flag chapters for review
- User approval required

**Layer 2: Pre-Writing Analysis**
- Scene-level breakdown
- Goal/Conflict/Outcome verification
- Detailed pattern scanning
- User approval if issues

**Layer 3: Writer Instructions**
- Mandatory quality checklist
- Explicit "avoid" list
- Show vs Tell commitment
- Flagged scenes highlighted

**Layer 4: Post-Writing Review**
- Scan actual prose
- Detect slipped patterns
- Offer auto-revision
- User decision point

### Detection Patterns

**Keyword Triggers:**
- "learns about" / "learned that"
- "is told" / "was told"
- "explained" / "described"
- "tavern" / "inn" / "bar"
- "overhears" / "eavesdropped"
- "conversation revealed"
- "mentor/sage/elder said"
- "as [character] listened"

**Structural Patterns:**
- Character goes to location → learns information
- NPC conveniently knows everything
- Stranger provides crucial exposition
- Mentor explains system/prophecy/history
- Character passive recipient of info
- No cost/risk to gaining knowledge

**Green Flags (GOOD patterns):**
- Character PURSUES goal actively
- Information has COST (risk, relationships, resources)
- Discovery through ACTION
- Character WRONG, corrects mistake
- Conflict WHILE learning
- Must NEGOTIATE/STEAL/DEDUCE
- Social/physical/emotional price paid

### Outline Approval Decision Tree

```

      │
      ↓
   Types?
      │
      ├─→ "YES" → Proceed to writing
      │

      │           ↓
      │         Re-show checkpoint
      │           │
      │           ↓
      │         Require new approval
      │           │
      │           └─→ (back to top)
      │
      └─→ Chapter numbers (e.g., "3, 5")
                  │
                  ↓

            - Suggest alternatives

                  ↓

                  ├─→ YES → Revise outline
                  │         │
                  │         ↓

                  │         │
                  │         └─→ (back to top)
                  │
                  └─→ NO → Show outline again
                            │
                            └─→ (back to top)
```

## Key Differences: YOLO vs In-Depth

| Aspect | YOLO Mode | In-Depth Mode |
|--------|-----------|---------------|
| **Planning Time** | 5-10 minutes | 30-60 minutes |
| **Character Detail** | Sketches (1 page) | Full profiles (2-3 pages) |
| **Outline Depth** | 3-act bullets | Detailed beats |
| **Trope Research** | 5 tropes, pick 1-2 | 10 tropes, pick 2-3 |
| **Name Options** | 5 per character | 10 per character |

## Workflow Recommendations

### When to Use YOLO Mode

- You want fast first draft
- You discover through writing
- You trust the process
- You're willing to revise later
- You want minimal decision points
- You prefer automatic flow

### When to Use In-Depth Mode

- You prefer detailed planning
- You want control between arcs
- You like reviewing before continuing
- You have complex world-building
- You're writing series/sequels
- You want to prevent major rewrites

### Hybrid Approach

**Not officially supported but possible:**
1. Start YOLO for Act 1
2. Review results
3. Switch to In-Depth for Act 2-3
4. More control where complexity increases

## Anti-Cookie-Cutter Safeguards Summary

**Total safeguards: 7 layers**

1. **Trope Research** - Avoid AI plot patterns
2. **Name Generator** - Prevent "Kael", "Elara", "Lyra"
3. **Goal/Conflict/Outcome** - Chapter-level structure
4. **Exposition Detection** - Automatic pattern scanning
5. **Outline Approval** - User review before writing (YOLO)
6. **Pre-Writing Analysis** - Scene verification
7. **Post-Writing Review** - Catch slipped patterns

**Result:** High-quality, non-formulaic storytelling even in fast YOLO mode.

3. Pre/post reviews still RUN but don't block

**Warnings given:** Multiple, strong

**Risk:** Cookie-cutter prose, major rewrites needed

1. `/write:chapter {N}` - Rewrite specific chapter

3. Reference STRUCTURAL-REVIEW.md findings
4. Spawn new writer with "fix exposition" focus

**Auto-revision option:** Offered at post-gen review
